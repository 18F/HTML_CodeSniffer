WCAG 2.0 Success Criterion 2.1.3: Keyboard (No Exception) (Level AAA)

Back to: Home - Level AAA Summary - Level AA Summary

Definition

All functionality of the content is operable through a keyboard interface without requiring specific timings for individual keystrokes.

Editor's Note:

No additional techniques are provided over and above Success Criterion 2.1.1. Instead, if analogue, time-dependent input is required, the content simply cannot meet AAA conformance.

More discussion from W3C: Understanding Success Criterion 2.1.3

Summary

This is a more restrictive version of Success Criterion 2.1.1.

No additional techniques are provided over and above that provided by Success Criterion 2.1.1, and any messages from techniques that apply there will be thrown as that Success Criterion, not this one.

The way the Understanding document for Success Criterion 2.1.3 is worded is sort-of an abandonment of all hope where analogue, time-dependent input is required: it essentially states that there is no way to make such input conformant with the AAA requirement.
