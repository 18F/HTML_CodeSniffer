Success Criterion 4.1.1: Parsing (Level A)

In content implemented using markup languages, elements have complete start and end tags, elements are nested according to their specifications, elements do not contain duplicate attributes, and any IDs are unique, except where the specifications allow these features.

Note: Start and end tags that are missing a critical character in their formation, such as a closing angle bracket or a mismatched attribute value quotation mark are not complete.

(From Understanding WCAG 2.0: Understanding Success Criterion 4.1.1)

Discussion

Valid vs. Well-formed

Keep in mind that "valid" and "well-formed" are two different things, at least in XML. In an XML document (including XHTML), it only needs to be well-formed to satisfy this Success Criterion - not necessarily valid (technique H75: Ensuring that Web pages are well-formed).

Replacements for well-formedness requirements in HTML

Because HTML serialisation does not have a concept of being "well-formed" so to speak, following technique H88: Using HTML according to spec is sufficient. Otherwise, at an absolute minimum, the following three techniques must be followed:

  • H74: Ensuring that opening and closing tags are used according to specification
  • H93: Ensuring that id attributes are unique on a Web page
  • H94: Ensuring that elements do not contain duplicate attributes

HTML_CodeSniffer is limited to information it can receive through the DOM, which will have likely processed out duplicate attributes, and certainly opening/closing tags in the generation process of the DOM tree. Therefore, it can only detect duplicate ID attributes. These errors are flagged as a Common Failure - Failure of Success Criterion 4.1.1 due to duplicate values of type ID.

A lack of failures of this kind can only be considered authoritative if a complete document is processed. A document fragment can be tested for duplicate IDs, but there is nothing to say that it will duplicate the IDs of elements outside of that fragment.

The failure F77 refers to "duplicate values of type ID". It is worded this way because it applies to any XML-based markup language, and ID attributes can be defined for other XML documents using either a DTD or XML Schema. This does not matter so much for HTML documents.
