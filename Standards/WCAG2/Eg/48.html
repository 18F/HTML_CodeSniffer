HTML_CodeSniffer Test Example 48

According to Success Criterion 1.1.1: Non-text Content, all non-text content, including embedded objects, should contain a text alternative. Although the applet element was deprecated in favour of object in HTML4 (it was introduced in HTML 3.2), it still remains a popular method of including Java applets.

Two text alternatives should be provided for applet elements: one in the body, and one as an alt attribute. The alt attribute is for those browsers who understand the applet element natively - Internet Explorer falls into this category - but cannot load Java. The body text is for those that will not understand it (such as Firefox) without a plugin, and thus would by default ignore the tag completely.

In this case, the alt attribute has not been provided.

Code

<applet code="lake.class" width="250" height="500">
    <p>Lake applet</p>
</applet>
