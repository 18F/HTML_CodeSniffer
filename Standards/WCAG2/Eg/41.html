According to Success Criterion 1.1.1: Non-text Content, all client-side image maps, using the map element, should have alt text (a short-text alternative) for its constituent area elements.

This example demonstrates an image map with two areas, both containing alt elements as specified. A notice is emitted to remind authors of the responsibility of ensuring the contents of the area element "serves the same purpose as the part of image map image referenced by the area element". In this example, these alternatives are probably not adequate.

Code

<map id="foomap">
    <area type="rect" coords="0,0,50,50" href="about.html" alt="Area 1" />
    <area type="rect" coords="50,50,100,100" href="screenshots.html" alt="Area 2" />
</map>
