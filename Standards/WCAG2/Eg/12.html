HTML_CodeSniffer Test Example 12

A one-to-one relationship should be able to be determined between IDs and elements with IDs. Therefore, controls with duplicate IDs are a failure of the specs of Success Criterion 4.1.1: Parsing. In XML serialisation, it would not be well-formed unless duplicate IDs are explicitly allowed.

This is marked specifically to a failure regarding duplicate IDs - F77: Failure of Success Criterion 4.1.1 due to duplicate values of type ID - particularly relating to HTML serialisation. However, this could also be attributed to a failure of Success Criterion 1.3.1, using failure F17: Failure of Success Criterion 1.3.1 and 4.1.1 due to insufficient information in DOM to determine one-to-one relationships (e.g., between labels with same id) in HTML.

Code

<form>
    <input type="text" id="control1" value="field 1" />
    <input type="text" id="control1" value="field 2" />
</form>
